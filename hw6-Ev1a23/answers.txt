המימוש public void push(Object o) – המימוש אינו תואם את החוזה, מכיוון ובחוזה אין הגבלה ש object O יהיה null, מה שיפר את האינווריאנטה של רשימה מקושרת, שלא ניתן להחזיר null מ – top.
דוגמא נגדית: push(null).
תיקון לחוזה: הוספת השורה הבאה
@pre: o!=null

המימוש public void pop()  - המימוש תואם את החוזה, הרשימה אינה ריקה. 
שמרנו את האיבר העליון שהוא זה שיוצא מהרשימה בפעולת ה – pop, ואותו גם החזרנו. 

המימוש public Object top() – המימוש תואם את החוזה, הרשימה אינה ריקה , ואסור להחזיר null מפעולה זו (האינווריאנטה שמצוינת לפני הקוד של המחלקה). 
לכן, בהכרח חייב להיות איבר שאינו null ברשימה, וכאשר נגיע אליו הוא זה שיוחזר.

המימוש public Boolean isEmpty() – המימוש אינו תואם את החוזה, לפי החוזה כאשר כמות האיברים גדולה מ-0 נחזיר true, אך המימוש מציין שנחזיר true אמ"מ כמות האיברים שווה ל-0. 
לכן, עבור הקלט [1,2] לפי החוזה אמור לחזור true, אך לפי המימוש יחזור false כי כמות האיברים היא 2, שונה מ-0.
דוגמא נגדית: isEmpty([1,2])
תיקון לחוזה: תיקון השורה
@post @return == true iff elements.size() ==  0
